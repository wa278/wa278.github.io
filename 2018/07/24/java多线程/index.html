线程安全性

核心在于管理共享的，可变的状态
当多个线程访问一个可变的变量时，如果没有合适的措施，就会产生错误，解决方案有两个：

  1. 修改状态为不可变对象
  2. 在访问变量时使用同步机制

    原子性操作

    不正确的进行原子性操作，容易发生竞态条件（Race Condition）,常见的竞态条件有如下两种：
    1. a++操作 实际上包含了三步：读，加，写。
    2. 先检查后执行，比如延迟初始化，if(a == null)
      要确保单个原子操作中更新所有有关的状态变量

      用锁来保护对象

  3. 同一个对象的读取都要同一把锁保护。
  4. 同一个不变性条件涉及的对象，都要用同一把锁。
